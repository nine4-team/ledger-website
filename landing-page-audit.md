# Landing Page Audit: Section Redundancy & Clarity Issues

## Executive Summary
The landing page has **significant messaging overlap** between Admin & Billing sections, unclear section purposes, and features categorized incorrectly. The core issue: **we're organizing by vague categories instead of the designer's actual workflow.**

---

**Section 2: "Professional reports without the spreadsheet nightmare"**

## What Each Section SHOULD Be About

Based on your transcript, the app supports a **linear project workflow:**

```
1. PLAN BUDGET ‚Üí 2. SOURCE/SHOP ‚Üí 3. INSTALL ‚Üí 4. BILL & CLOSE
```

### Recommended Section Structure:

#### **BUDGET TRACKING** (Pre-purchase)
**Purpose:** Help designers plan spending and avoid budget blowouts BEFORE they buy
- Real-time budget visibility
- Category breakdown (furniture vs moving vs storage)
- Confidence when shopping

#### **INVENTORY & RECEIPT MANAGEMENT** (During sourcing)
**Purpose:** Track what you've bought and keep receipts organized as you source
- Centralized item list across projects
- Receipt capture and storage
- Purchase attribution (who paid: you or client)

#### **INSTALL DAY TOOLS** (During install)
**Purpose:** Handle the chaos when you're on-site with clients
- One-tap return marking
- Instant receipt lookup
- Quick item reference

#### **CLIENT BILLING & DOCUMENTATION** (Post-project)
**Purpose:** Get reimbursed and deliver professional documentation
- **Reimbursement invoicing** - Bill clients for items you purchased on their behalf
- **Project documentation** - Generate itemized reports for clients/property managers

## Specific Redundancy Examples

### Example 1: "Automatic generation" messaging appears 4+ times
- Hero: "handles the chaos"
- Budget section: "No math, no guessing"
- Invoicing: "creates professional invoices for you automatically"
- Reports: "click a button and it's ready"
- Install Day: "you'll have a list of every item"

**Fix:** Each section should emphasize the UNIQUE pain point it solves, not just "we automate it."

### Example 2: "No spreadsheets" appears 3 times
- Hero subheadline
- Problem section
- Reports section

**Fix:** Move this to ONLY the hero/problem sections. Individual features should highlight specific outcomes, not repeat the broad promise.

### Example 3: Receipt management appears in 2 disconnected places
- Install Day section: "Find any receipt in seconds"
- Category card: Receipt management benefits

**Fix:** Consolidate receipt features into ONE section about sourcing/inventory management.

## The Root Cause

**You're organizing by department names (Admin, Billing, Planning) instead of designer workflows.**

Designers don't think: "Now I'm doing admin tasks"  
They think: "I'm about to buy 40 things at HomeGoods and I need to know if I can afford it"

---

## Recommended Action Plan

### Phase 1: Restructure Sections by Workflow Phase
1. **Budget Planning** (before shopping)
2. **Sourcing & Shopping** (while buying)
3. **Install Execution** (on-site with client)
4. **Billing & Closeout** (getting paid and wrapping up)

### Phase 2: Split the "Admin & Billing" Mega-Section
- **Section A: "Get Reimbursed Fast"** ‚Üí Focus on billing clients for items you purchased
- **Section B: "Deliver Professional Documentation"** ‚Üí Focus on final project deliverables

### Phase 3: Clarify Feature Purposes
- Budget sections should ONLY talk about money/spending decisions
- Inventory sections should focus on remembering/organizing items
- Billing sections should distinguish between "getting paid" vs "delivering documentation"

### Phase 4: Eliminate Redundant Messaging
- Remove "automatic/no spreadsheets" from individual sections
- Give each section ONE clear job to do
- Make sure eyebrows accurately reflect the section content

## Questions to Resolve

1. **Are invoicing and documentation actually the same feature?** Or can users generate two different types of outputs?

2. **Should "tracking who purchased what" live in:**
   - Install Day (helps decide what to keep/return)
   - Billing (helps know who owes whom)
   - Sourcing (tracked at time of purchase)

3. **Is budget tracking primarily about:**
   - Pre-purchase planning? ("Can I afford this before I buy?")
   - Post-purchase awareness? ("Where did the money go?")
   - Both?

4. **What's the difference between these features from the user's perspective:**
   - "Generate invoice" 
   - "Create itemized breakdown"
   - "Deliver documentation"

---

## Bottom Line

Your landing page describes ~7-8 features but organizes them into categories that:
- Overlap heavily (Admin vs Billing)
- Don't match user mental models (Budget vs Inventory)
- Repeat the same value props ("automatic," "no spreadsheets")

**Solution:** Reorganize by the designer's project timeline, give each section one clear job, and ensure the messaging for each feature highlights a UNIQUE pain point.
